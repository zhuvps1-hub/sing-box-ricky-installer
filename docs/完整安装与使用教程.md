# iWAN Gateway 完整安装、升级与使用教程

> 适用于公开部署、个人 VPS 和已有 sing-box/iWAN 环境。本文以公开通用版 **v6.7** 为准。

项目仓库：`zhuvps1-hub/sing-box-ricky-installer`

---

## 1. 项目用途

本项目将 iWAN 中转、sing-box 多落地分流和 Web 管理面板组合在一起：

```text
手机 / Panabit / Linux iWAN 客户端
                │
                ▼
        VPS 上的 iWAN 入站
                │
                ▼
             sing-box
       ┌────────┼────────┐
       ▼        ▼        ▼
     direct    节点 A    节点 B
    国内网站   指定业务   默认出口
```

Web 面板可管理：

- iWAN 入站监听地址、端口、地址池、MTU、用户名和密码；
- Shadowsocks 落地节点新增、编辑、删除、测速和一键导入；
- Netflix、ChatGPT/Claude/Gemini、YouTube、Telegram、其他流量独立出口；
- mosdns 完整 YAML、备份恢复和规则文件；
- sing-box、mosdns、面板日志和网络状态。

### 公开版原则

1. 不预设任何人的 iWAN 用户名；
2. 不预设 HKT、SG、JP 等落地节点；
3. 不在脚本中保存业务密码；
4. 新机器安装后直接登录 Web 面板初始化；
5. 面板升级不覆盖底层服务；
6. 所有配置修改都先校验、备份，失败自动回滚。

## 2. 脚本说明

| 文件 | 用途 | 是否修改底层 |
|---|---|---|
| `install.sh` | 全新 VPS 一键安装 sing-box 核心、空白配置和 Web 面板 | 是 |
| `install-web.sh` | 已有环境只安装或升级 Web 面板 | 否 |
| `install-panel-stable.sh` | 面板下载、校验、候选测试、原子升级和回滚 | 只管理面板 |
| `panel-release.json` | 固定版本、不可变提交和每个文件的 Git Blob 校验值 | 否 |
| `panel/` | Python、HTML、CSS 和 JavaScript 源码 | 否 |
| `.github/workflows/panel-ci.yml` | 语法、自检、组装和发布清单校验 | 否 |

### `install.sh` 做什么

全新机器执行后会：

1. 安装依赖；
2. 下载并校验 sing-box `with_iwan` 二进制；
3. 安装 `sing-box.service`；
4. 新机器创建只有 `direct` 的空白配置；
5. 已有配置时保留原文件并创建备份；
6. 自动调用 Web 面板安装器；
7. 引导用户登录 Web 后设置自己的 iWAN 和节点。

它不会再询问 HKT、SG、iWAN 用户名或 Shadowsocks 密码。

### `install-web.sh` 做什么

- 只安装或升级 Web 面板；
- 不安装 sing-box；
- 不覆盖 `/etc/sing-box/config.json`；
- 不安装 mosdns；
- 候选版本验证成功前不停止旧面板；
- 正式启动失败自动恢复旧版。

## 3. 系统要求

### 支持

- Linux amd64 / x86_64；
- systemd；
- Debian、Ubuntu、Rocky Linux、AlmaLinux；
- root 用户；
- Python 3.10 或更高版本；
- 能访问 GitHub Release、raw.githubusercontent.com 或 jsDelivr。

### 暂不支持

- ARM / ARM64；
- OpenWrt；
- Alpine/OpenRC；
- 非 systemd 系统。

## 4. 全新 VPS 一键安装

使用 root 登录 VPS，执行：

```bash
bash <(curl -fsSL "https://raw.githubusercontent.com/zhuvps1-hub/sing-box-ricky-installer/main/install.sh?ts=$(date +%s)")
```

安装过程中只需要设置 Web 面板登录账号和密码。

不会要求输入：

- iWAN 用户名；
- iWAN 业务密码；
- HKT、SG、JP 节点；
- Shadowsocks 节点密码；
- Netflix、AI、YouTube、Telegram 出口。

### 初始 sing-box 配置

新机器会生成：

```json
{
  "log": {
    "level": "info",
    "timestamp": true
  },
  "inbounds": [],
  "outbounds": [
    {
      "type": "direct",
      "tag": "direct"
    }
  ],
  "route": {
    "rules": [],
    "final": "direct"
  }
}
```

此时 sing-box 可以作为空白底层运行，但尚未开放 iWAN 入口，也没有任何落地节点。

### 默认面板地址

```text
http://你的VPS公网IP:8088
```

云服务商安全组至少需要放行：

```text
8088/TCP
```

建议只允许自己的公网 IP 访问面板。

## 5. 首次登录后的初始化

### 第一步：创建 iWAN 入口

打开底部 **iWAN** 页面，填写：

- 监听地址：通常使用 `::`；
- 监听端口：例如 `8000`，可按实际修改；
- 地址池：例如 `10.10.10.0/24`；
- MTU：通常 `1400`；
- 用户名：填写自己的 iWAN 用户名；
- 密码：填写自己的 iWAN 密码。

点击：

```text
保存并重连
```

如果当前配置中还没有 iWAN inbound，面板会自动创建：

```json
{
  "type": "iwan",
  "tag": "iwan-in",
  "listen": "::",
  "listen_port": 8000,
  "address_pool": "10.10.10.0/24",
  "mtu": 1400,
  "users": [
    {
      "username": "用户填写的用户名",
      "password": "用户填写的密码"
    }
  ]
}
```

随后面板会：

1. 运行 `sing-box check`；
2. 备份旧配置；
3. 写入新配置；
4. 重启 sing-box；
5. 检查服务状态；
6. 失败时自动恢复旧配置。

配置完成后，需要在云安全组中放行实际 iWAN 端口的 TCP 和 UDP，例如：

```text
8000/TCP
8000/UDP
```

### 第二步：导入自己的节点

打开 **节点** 页面，可选择：

- 新增节点；
- 粘贴一个或多个 `ss://`；
- 导入节点 JSON；
- 导入完整 sing-box `outbounds`。

支持的常见 Shadowsocks 加密方式包括：

```text
aes-128-gcm
aes-256-gcm
chacha20-ietf-poly1305
xchacha20-ietf-poly1305
2022-blake3-aes-128-gcm
2022-blake3-aes-256-gcm
2022-blake3-chacha20-poly1305
```

导入流程：

```text
粘贴内容
→ 自动解析
→ 预览节点
→ 处理同名节点
→ 确认加入
→ 后台自动保存
```

### 第三步：设置独立分流

打开 **分流** 页面，分别选择：

```text
国内网站        → direct
Netflix         → 自选节点
ChatGPT/Claude  → 自选节点
YouTube         → 自选节点
Telegram        → 自选节点
其他流量        → 自选默认节点
```

每一类完全独立，不会自动跟随其他分类。

节点和分流修改后会可靠自动保存：

1. 浏览器生成唯一请求号；
2. 服务器先将待处理配置原子写入磁盘；
3. 立即确认接收；
4. 后台合并连续修改；
5. 校验、备份、写入并重启 sing-box；
6. 网络短暂断开时自动确认和重试；
7. 失败才显示提示并自动回滚。

成功时不会弹出多余提示，也不会锁住页面。

已经有底层配置时执行：

- mosdns 配置；

## 7. 面板升级机制

面板稳定安装器采用：

```text
固定提交 Raw 下载
→ Git Blob 逐文件校验
→ Python 语法检查
→ 内置自检
→ 隔离候选进程
→ 健康接口测试
→ 登录测试
→ 配置采样测试
→ 原子切换 current 软链接
→ 正式健康检查
→ 失败恢复旧版
```

升级命令：

```bash
bash <(curl -fsSL "https://raw.githubusercontent.com/zhuvps1-hub/sing-box-ricky-installer/main/install-web.sh?ts=$(date +%s)")
```

带时间戳是为了绕过 CDN 和浏览器缓存。

## 8. mosdns 页面

服务器已安装 mosdns 时，DNS 页面支持：

- 编辑 `/etc/mosdns/config.yaml`；
- 保存并重启；
- 自动备份；
- 启动失败自动恢复；
- 查看插件、监听和上游摘要；
- 管理 YAML、TXT、LIST、HOSTS 等辅助文件；
- 手动备份；
- 恢复历史备份；
- 单独重启 mosdns。

本项目的 `install.sh` 不自动安装 mosdns。没有 mosdns 时，DNS 页面显示不可用，其他功能不受影响。

## 10. 常用命令

### 查看服务状态

```bash
systemctl status sing-box iwan-gateway mosdns --no-pager
```

### 查看面板健康接口

```bash
curl -s http://127.0.0.1:8088/healthz
```

### 检查 sing-box 配置

```bash
/usr/local/bin/sing-box check -c /etc/sing-box/config.json
```

### 查看 sing-box 日志

```bash
journalctl -u sing-box -n 100 --no-pager
journalctl -u sing-box -f
```

### 查看面板日志

```bash
journalctl -u iwan-gateway -n 100 --no-pager
journalctl -u iwan-gateway -f
```

### 重启服务

```bash
systemctl restart sing-box
systemctl restart iwan-gateway
systemctl restart mosdns
```

### 查看当前配置

```bash
cat /etc/sing-box/config.json
```

## 11. 防火墙和安全组

至少放行：

```text
8088/TCP       Web 面板
实际端口/TCP   iWAN 入口
实际端口/UDP   iWAN 入口
```

例如 iWAN 使用 8000：

```text
8000/TCP
8000/UDP
```

安全建议：

- 面板 8088 只允许自己的公网 IP；
- 公开使用时通过 Nginx、Caddy 等反向代理提供 HTTPS；
- 使用至少 8 位面板密码；
- 不要在公开 issue、截图或教程中展示节点密码和 iWAN 密码；
- 定期备份 `/etc/sing-box`、`/etc/iwan-gateway` 和 `/etc/mosdns`。

## 12. 备份

```bash
mkdir -p /root/iwan-backup

cp -a /var/lib/iwan-gateway /root/iwan-backup/
```


## 14. 常见问题

### 安装后为什么 iWAN 显示未识别？

公开版新机器初始没有写入任何 iWAN 账号，这是正常现象。打开 iWAN 页面填写信息并点击“保存并重连”即可自动创建入口。

### 为什么节点页面为空？

公开版不会预设 HKT、SG 或其他节点。请新增或导入自己的节点。

### 为什么修改分流时网页会短暂波动？

后台需要重启 sing-box 使配置生效。如果访问面板的流量本身经过该中转，连接可能短暂抖动。可靠保存会自动确认和重试，不会重复应用。

### 保存失败会不会破坏原配置？

不会。配置先经过 `sing-box check`，失败不覆盖；启动失败会恢复备份并重新启动旧配置。

### 已有配置会不会被 `install.sh` 覆盖？

不会。检测到 `/etc/sing-box/config.json` 后会保留原配置，并额外创建时间戳备份。

### 面板升级会不会重启 sing-box？

安装或升级面板本身不会。只有用户在网页中修改并应用业务配置时，才会重启 sing-box。

- `install.sh` 中没有真实用户名；
- 没有 HKT、SG、JP 固定节点；
- 没有 Shadowsocks 密码；
- 没有 iWAN 密码；
- README 使用空白首次初始化流程；
- `panel-release.json` 指向不可变提交；
- GitHub Actions 全部通过；
- 新机器安装测试可以进入 Web 并首次创建 iWAN inbound。
