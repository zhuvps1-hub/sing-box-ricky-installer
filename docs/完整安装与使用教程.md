# iWAN Gateway 完整安装、升级与使用教程

> 适用于公开部署、个人 VPS 和已有 sing-box/iWAN 环境。本文以当前稳定面板 **v6.6.0** 为准。
>
> 项目仓库：`zhuvps1-hub/sing-box-ricky-installer`

---

## 1. 项目解决什么问题

本项目把 iWAN 中转、sing-box 多落地分流和 Web 管理面板组合在一起：

```text
手机 / Panabit / Linux iWAN 客户端
                │
                ▼
       VPS 上的 iWAN 入站
                │
                ▼
             sing-box
       ┌────────┼────────┐
       ▼        ▼        ▼
     direct   SG/JP     HK/HKT
    国内网站   指定业务    默认出口
```

面板可管理：

- iWAN 入站监听端口、地址池、MTU、用户名和密码；
- Shadowsocks 落地节点的新增、编辑、删除、测速和导入；
- Netflix、ChatGPT/Claude/Gemini、YouTube、Telegram、其他流量的独立出口；
- mosdns 完整 YAML、备份恢复和辅助规则文件；
- sing-box、mosdns、面板日志和网络状态。

### 核心设计原则

1. **底层服务和 Web 面板分离**：升级面板不会覆盖 sing-box 或 mosdns。
2. **节点和分流可靠自动保存**：先在服务器安全暂存，再由后台校验和应用。
3. **所有变更可回滚**：配置校验失败不会覆盖现有配置，服务启动失败自动恢复备份。
4. **公开可用**：教程和脚本不依赖某个用户的账号、密码或固定节点。

## 2. 仓库脚本说明

| 文件 | 用途 | 是否修改底层 |
|---|---|---|
| `install.sh` | 全新 VPS 安装 sing-box iWAN 核心和初始分流 | 是 |
| `install-web.sh` | Web 面板统一入口，负责下载稳定安装器 | 否 |
| `install-panel-stable.sh` | 面板的下载、校验、候选测试、原子升级和回滚 | 只管理面板 |
| `panel-release.json` | 固定面板版本、不可变提交和每个文件的 Git Blob 校验值 | 否 |
| `panel/` | Python 后台、HTML、CSS、JavaScript 源码 | 否 |
| `.github/workflows/panel-ci.yml` | 自动执行语法、自检、组装和发布清单校验 | 否 |

### 两个安装器的区别

- `install.sh`：部署 sing-box iWAN 核心。会生成 `/etc/sing-box/config.json`，安装并启动 `sing-box.service`。
- `install-web.sh`：只安装或升级 Web 面板。不会安装、覆盖或重启 sing-box、mosdns。

> mosdns 不是由 `install.sh` 自动安装的。服务器没有 mosdns 时，面板的 DNS 页面只会显示不可用；其他功能不受影响。

## 3. 系统要求

### sing-box 核心安装器

- Linux amd64 / x86_64；
- 使用 systemd；
- Debian、Ubuntu、Rocky Linux、AlmaLinux 等；
- root 用户；
- 能访问 GitHub Release；
- 当前脚本不支持 ARM、OpenWrt、Alpine/OpenRC。

### Web 面板

- Python 3.10 或更高版本；
- curl、CA 证书；
- systemd；
- root 用户；
- 默认端口 `8088/TCP`。

查看系统环境：

```bash
uname -m
cat /etc/os-release
python3 --version || true
systemctl --version | head -n 1
```

## 4. 部署前准备

准备以下信息：

```text
iWAN 监听端口，例如 8000

iWAN 地址池，例如 10.10.10.0/24

默认落地节点：服务器、端口、加密方式、密码
```

### 安全提醒

- 不要把真实密码提交到公开 GitHub 仓库；
- 不要把包含密码的 `/etc/sing-box/config.json` 发到公开群聊；
- 面板默认使用 HTTP，建议限制 `8088` 只允许自己的公网 IP；
- 公开访问时应使用 Nginx/Caddy 反向代理和 HTTPS；
- 面板密码至少 12 位，避免与 iWAN、SSH、节点密码相同。

## 5. 方案一：全新 VPS 完整安装

全新服务器分两步：

```text
第一步：安装 sing-box iWAN 核心
第二步：安装 Web 面板
```

### 5.1 交互式安装 sing-box 核心

使用 root 登录 VPS：

```bash
ssh root@你的VPS公网IP
```

执行：

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/zhuvps1-hub/sing-box-ricky-installer/main/install.sh)
```

脚本会提示输入：

1. iWAN 用户密码；
2. 默认 HKT 落地密码；
3. SG 落地密码。

安装器会自动完成：

- 安装 curl、CA 证书和 tar；
- 下载 `sing-box-1.13.13-rickyhao.22`；
- 校验安装包 SHA-256；
- 检查二进制包含 `with_iwan` 标签；
- 备份旧配置；
- 生成 iWAN 入站、两个 Shadowsocks 出站和初始分流；
- 执行 `sing-box check`；
- 创建并启动 `sing-box.service`；
- 尝试通过 UFW/firewalld 放行 iWAN 端口。

### 5.2 公共部署：自定义所有参数

`install.sh` 中带有示例默认地址。公开给其他人使用时，建议显式传入自己的参数，不要依赖默认值。

先安全读取密码：

```bash
read -rsp 'iWAN 密码：' IWAN_PASSWORD; echo
read -rsp '默认落地密码：' HKT_PASSWORD; echo
read -rsp '业务落地密码：' SG_PASSWORD; echo
export IWAN_PASSWORD HKT_PASSWORD SG_PASSWORD
```

设置其他参数：

```bash
export IWAN_PORT='8000'
export IWAN_USERNAME='your-iwan-user'
export IWAN_ADDRESS_POOL='10.10.10.0/24'

export HKT_SERVER='default.example.com'
export HKT_PORT='24895'
export HKT_METHOD='2022-blake3-aes-128-gcm'

export SG_SERVER='service.example.com'
export SG_PORT='22222'
export SG_METHOD='aes-128-gcm'
```

开始安装：

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/zhuvps1-hub/sing-box-ricky-installer/main/install.sh)
```

安装后清理当前 shell 中的密码变量：

```bash
unset IWAN_PASSWORD HKT_PASSWORD SG_PASSWORD
```

### 5.3 安装 Web 面板

交互式安装：

```bash
bash <(curl -fsSL "https://raw.githubusercontent.com/zhuvps1-hub/sing-box-ricky-installer/main/install-web.sh?ts=$(date +%s)")
```

首次安装会询问：

- 面板端口，默认 `8088`；
- 面板用户名，默认 `admin`；
- 面板密码，至少 8 位。

安装完成后访问：

```text
http://你的VPS公网IP:8088
```

### 5.4 非交互式安装面板

适合自动化、云初始化或批量部署：

```bash
export PANEL_PORT='8088'
export PANEL_USER='admin'
export PANEL_PASSWORD='请替换为强密码'

bash <(curl -fsSL "https://raw.githubusercontent.com/zhuvps1-hub/sing-box-ricky-installer/main/install-web.sh?ts=$(date +%s)")

unset PANEL_PASSWORD
```

## 8. 面板功能使用

### 8.1 登录与记住登录

登录页可勾选“记住登录”。面板保存的是签名登录凭证，不会在 LocalStorage 中保存明文密码。

- 普通会话适合临时设备；
- 记住登录适合自己的手机或 iPad；
- 公共设备不要勾选；
- 点击退出会清除登录凭证。

### 8.2 节点管理

节点页支持：

- 新增、编辑、删除 Shadowsocks 节点；
- TCP 延迟检测；
- 一键导入 `ss://`；
- 导入节点 JSON 数组；
- 导入完整 sing-box 配置中的 `outbounds`。

支持的常用加密方式：

```text
aes-128-gcm
aes-256-gcm
chacha20-ietf-poly1305
xchacha20-ietf-poly1305
2022-blake3-aes-128-gcm
2022-blake3-aes-256-gcm
2022-blake3-chacha20-poly1305
```

导入示例：

```text
ss://BASE64内容#节点名称
ss://BASE64内容#另一个节点
```

或者粘贴：

```json
[
  {
    "type": "shadowsocks",
    "tag": "sg-landing",
    "server": "sg.example.com",
    "server_port": 443,
    "method": "aes-128-gcm",
    "password": "请填写节点密码"
  }
]
```

### 8.3 业务分流

分流页的每一项完全独立：

```text
国内网站                  → direct
Netflix                   → 自选节点
ChatGPT / Claude / Gemini → 自选节点
YouTube                   → 自选节点
Telegram                  → 自选节点
其他流量                  → 自选默认节点
```

AI 不会自动跟随 Netflix，YouTube、Telegram 和默认出口也不会互相联动。

面板生成规则时，会先执行 HTTP、TLS、QUIC 域名识别，再分别匹配业务规则。Telegram 的域名和 IP 段会拆成独立规则，避免不同条件被同时要求命中。

> ECH 会加密 TLS ClientHello 中的域名。目标网站启用 ECH 后，仅依靠域名嗅探可能无法分流，需要增加 IP 规则或其他识别方式。

### 8.4 v6.6 可靠自动保存

节点和分流没有全局“保存”按钮，修改后自动处理：

1. 浏览器等待约 1 秒，合并连续操作；
2. 生成唯一请求号；
3. 服务器把待应用配置原子写入：
   `/var/lib/iwan-gateway/autosave-pending.json`；
4. 服务器立即确认接收；
5. 后台等待约 5 秒，继续合并新修改；
6. 执行 `sing-box check`；
7. 备份旧配置；
8. 原子替换配置并重启 sing-box；
9. 启动失败时恢复旧配置；
10. 成功时不弹窗，只有失败或回滚才提示。

同一个请求号重复提交不会执行两次。面板进程重启后，会自动恢复未完成的暂存任务。

> 重启 sing-box 时 iWAN 隧道可能短暂中断。如果当前手机访问面板也经过这条隧道，网页请求可能短暂断开，但 v6.6 会静默重试，不应把网络波动误报为保存失败。

### 8.5 iWAN 设置

在 iWAN 页面修改：

- 监听地址；
- 监听端口；
- 地址池；
- MTU；
- 用户名；
- 密码。

密码留空表示保留原密码。点击页面内的“保存并重连”后：

```text
校验配置 → 备份 → 写入 → 重启 sing-box → 检查状态 → 失败回滚
```

iWAN 不使用自动保存，避免输入到一半就重启隧道。

### 8.6 DNS / mosdns

DNS 页面支持：

- 读取和编辑 `/etc/mosdns/config.yaml`；
- 保存前备份；
- 保存并重启 mosdns；
- 启动失败自动恢复；
- 手动备份和历史恢复；
- 管理 `/etc/mosdns` 下的 YAML、TXT、LIST、HOSTS 等文本文件；
- 限制路径越界、二进制文件和超过 1 MB 的文件。

DNS 配置使用独立“保存并重启”，不会与 sing-box 的节点/分流自动保存混在一起。

### 8.7 日志与网络

日志页可查看：

```text
sing-box
mosdns
iwan-gateway
```

网络状态按需读取：

```bash
ip route show
ss -lntup
```

## 12. 故障排查

### 12.1 面板打不开

```bash
systemctl status iwan-gateway --no-pager
journalctl -u iwan-gateway -n 100 --no-pager
ss -lntp | grep 8088
curl -v http://127.0.0.1:8088/healthz
```

本机 healthz 正常、外网打不开，通常是云安全组或系统防火墙未放行 `8088/TCP`。

### 12.2 iWAN 客户端连接不上

检查：

```bash
systemctl status sing-box --no-pager
ss -lntup | grep 8000
journalctl -u sing-box -n 100 --no-pager
```

确认客户端和服务端的端口、用户名、密码完全一致，并同时放行 TCP 和 UDP。

### 12.3 分流显示已选节点，但实际出口不对

检查生成规则：

```bash
python3 - <<'PY'
import json
p='/etc/sing-box/config.json'
with open(p,encoding='utf-8') as f:
    c=json.load(f)
for i,r in enumerate(c.get('route',{}).get('rules',[])[:20],1):
    print(i,json.dumps(r,ensure_ascii=False))
print('final =',c.get('route',{}).get('final'))
PY
```

应该先看到：

```json
{"action":"sniff","sniffer":["http","tls","quic"]}
```

再看到各业务独立的 `action: route` 规则。

### 12.4 修改后出现短暂断网

这是 sing-box 重启导致 iWAN 隧道短暂重连。v6.6 会先暂存并立即确认，再后台重启。通常几秒内恢复。

查看：

```bash
journalctl -u sing-box --since '-2 min' --no-pager
journalctl -u iwan-gateway --since '-2 min' --no-pager
```

### 12.5 自动保存失败

检查暂存文件和日志：

```bash
ls -l /var/lib/iwan-gateway/autosave-pending.json 2>/dev/null || true
journalctl -u iwan-gateway -n 150 --no-pager
journalctl -u sing-box -n 150 --no-pager
sing-box check -c /etc/sing-box/config.json
```

常见原因：

- 节点密码为空；
- 删除了仍被分流使用的节点；
- 加密方式与服务端不一致；
- sing-box 配置包含当前版本不支持的字段；
- `/etc/sing-box` 无法写入；
- 服务名称不是 `sing-box`。

### 12.6 GitHub 下载失败或 403

当前稳定安装器不使用 GitHub API，使用固定提交 Raw 下载，并有 jsDelivr 备用源。重试：

```bash
bash <(curl -fsSL "https://raw.githubusercontent.com/zhuvps1-hub/sing-box-ricky-installer/main/install-web.sh?ts=$(date +%s)")
```

仍失败时检查 VPS DNS、时间和 HTTPS：

```bash
date
getent hosts raw.githubusercontent.com
curl -I https://raw.githubusercontent.com
```

### 12.7 mosdns 保存失败

```bash
systemctl status mosdns --no-pager
journalctl -u mosdns -n 100 --no-pager
ls -l /etc/mosdns/config.yaml
```

YAML 缩进必须使用空格，不能使用 Tab。

## 13. 备份与恢复

### 手动备份关键文件

```bash
BACKUP="/root/iwan-backup-$(date +%Y%m%d-%H%M%S).tar.gz"
tar -czf "$BACKUP" \
  /etc/sing-box \
  /etc/iwan-gateway \
  /var/lib/iwan-gateway \
  /etc/mosdns 2>/dev/null || true
echo "$BACKUP"
```

### 恢复 sing-box 配置

```bash
cp -a /etc/sing-box/backups/你的备份文件.json /etc/sing-box/config.json
sing-box check -c /etc/sing-box/config.json
systemctl restart sing-box
```

### 查看旧面板版本

```bash
ls -1 /opt/iwan-gateway-panel/releases
readlink -f /opt/iwan-gateway-panel/current
```

正常升级失败时安装器会自动回滚，不建议手动修改 `current` 软链接。

## 15. 公开发布和二次开发建议

1. 不要在脚本、README、Issue 或截图中提交真实密码；
2. 示例服务器统一使用 `example.com`、`192.0.2.0/24` 等保留地址；

4. 每个发布文件必须记录正确 Git Blob SHA；
5. 修改面板模块后必须运行完整 CI；
6. 发布清单通过后再让用户执行安装命令；
7. 不要在候选版本测试前停止旧面板；
8. 面板升级脚本不得安装、覆盖或重启 sing-box/mosdns；
9. 核心配置变更必须先校验、备份，失败必须回滚；
10. 节点和分流自动保存需要幂等请求号，不能仅靠增加超时时间。
