# iWAN Gateway v7.1.2 部署与安全指南

v7.1.2 将面板拆分为两个进程：

- `iwan-gateway.service`：以专用非特权用户运行，默认监听 `0.0.0.0:8088`，便于通过公网 IP 直接访问。
- `iwan-gateway-helper.service`：以 root 运行，只接受本机 Unix Socket 请求，并通过 `SO_PEERCRED` 校验调用者 UID。

浏览器提交的 JSON 不会直接复制进 sing-box。Web 进程先转成稳定配置模型，helper 再执行校验、备份、原子替换、重启、健康检查和失败回滚。

## 1. 安装或升级

使用经过固定提交和 SHA-256 校验的稳定引导脚本：

```bash
curl -fsSL https://raw.githubusercontent.com/zhuvps1-hub/sing-box-ricky-installer/main/install-panel-stable.sh \
  -o /root/install-panel-stable.sh
bash /root/install-panel-stable.sh
```

稳定引导脚本只负责下载固定提交中的 v7.1.2 安装器并核对 SHA-256；正式安装器还会继续验证：

1. 固定 Ed25519 公钥的 SHA-256 指纹。
2. `panel-release.json` 的 Ed25519 签名。
3. 每个发布文件的 SHA-256。
4. 每个带 `git_blob` 文件的 Git Blob SHA-1。
5. 新版本独立自检和无旧版本栈依赖。
6. 两个 systemd 服务启动后的 `/healthz`。

安装器还会递归修复 `/var/lib/iwan-gateway` 的所有者和权限，避免旧 `panel.db` 导致 `attempt to write a readonly database`。健康检查失败时，`current` 软链接会自动切回上一个新架构版本；首次从旧版升级失败时会恢复旧 systemd 服务快照。

安装完成后访问：

```text
http://服务器公网IP:8088
```

云厂商安全组必须放行 `TCP 8088`。安装器会尝试自动放行 UFW 或 firewalld，但无法代替云安全组设置。

## 2. 公网访问与防火墙

v7.1.2 按当前项目使用需求默认开放公网监听。建议至少采取以下措施：

- 使用强管理员密码；
- 在云安全组中只允许自己的固定公网 IP 访问 `8088/TCP`；
- 不要将面板密码与 iWAN 节点密码设置为相同内容；
- 长期使用应配置 HTTPS。

UFW 示例：

```bash
ufw allow from 你的管理公网IP to any port 8088 proto tcp
ufw status numbered
```

临时允许所有来源测试：

```bash
ufw allow 8088/tcp
```

测试结束后应改为仅允许自己的管理 IP。

需要恢复仅本机监听时，编辑：

```text
/etc/iwan-gateway/gateway.env
```

设置：

```text
PANEL_BIND=127.0.0.1
IWAN_ALLOW_PUBLIC_BIND=0
```

然后执行：

```bash
systemctl restart iwan-gateway
```

## 3. 记住登录状态

登录页面提供“记住登录状态（30 天）”选项。该功能：

- 不在浏览器保存明文密码；
- 使用服务端 HMAC 签名 Cookie；
- Cookie 与当前 `auth.json` 指纹绑定；
- 修改管理员密码后，旧的长期登录 Cookie 自动失效；
- 主动退出会同时清除短期和长期 Cookie。

使用明文 HTTP 时，网络中间人仍可能窃取 Cookie，因此长期登录应配合 HTTPS 使用。

## 4. Caddy HTTPS（推荐）

复制模板：

```bash
cp deploy/caddy/Caddyfile.example /etc/caddy/Caddyfile
```

把 `panel.example.com` 替换为实际域名，然后执行：

```bash
caddy validate --config /etc/caddy/Caddyfile
systemctl reload caddy
```

Caddy 会自动申请和续签证书，并把真实协议与来源地址传给面板。HTTPS 生效后可关闭公网 `8088/TCP`，只保留 80/443。

## 5. Nginx HTTPS

复制模板：

```bash
cp deploy/nginx/iwan-gateway.conf.example /etc/nginx/sites-available/iwan-gateway.conf
ln -s /etc/nginx/sites-available/iwan-gateway.conf /etc/nginx/sites-enabled/iwan-gateway.conf
```

修改域名和证书路径，再执行：

```bash
nginx -t
systemctl reload nginx
```

## 6. 服务与日志

```bash
systemctl status iwan-gateway-helper --no-pager
systemctl status iwan-gateway --no-pager
journalctl -u iwan-gateway-helper -u iwan-gateway -n 200 --no-pager
curl -fsS http://127.0.0.1:8088/healthz
ss -lntp | grep ':8088'
```

正常公网监听应显示：

```text
0.0.0.0:8088
```

审计日志默认位于：

```text
/var/lib/iwan-gateway/audit.jsonl
```

日志会屏蔽密码、令牌、Authorization、Shadowsocks 链接等敏感内容，并进行容量轮换。

## 7. 回滚

回滚到最近一个新架构历史版本；没有历史版本时，会恢复最近的旧版服务快照：

```bash
bash /root/install-panel-stable.sh --rollback
```

回滚到指定新架构版本目录名：

```bash
bash /root/install-panel-stable.sh --rollback 7.1.2-7f8fd3dce1ea
```

首次升级保存的旧版服务快照可按目录名恢复：

```bash
ls -1 /opt/iwan-gateway/legacy/
bash /root/install-panel-stable.sh --rollback legacy-20260719-120000
```

版本与旧版快照目录：

```text
/opt/iwan-gateway/releases/
/opt/iwan-gateway/legacy/
```

配置备份位于：

```text
/etc/sing-box/backups/
/etc/mosdns/backups/
```

## 8. 发布私钥管理

仓库只保存 Ed25519 公钥。发布私钥必须：

- 不提交到 GitHub；
- 存放在离线密码库、硬件安全设备或受保护的 CI Secret 中；
- 只用于签署规范化后的 `panel-release.json`；
- 泄露时立即生成新密钥、更新仓库公钥和安装器固定指纹，再发布密钥轮换版本。

签名验证：

```bash
python3 tools/release_manifest.py verify panel-release.json release/iwan-release-ed25519-public.pem
```
