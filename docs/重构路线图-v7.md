## 1. 故障根因

历史报错：

```text
inbounds[0].address: json: unknown field "address"
```

旧面板把通用 TUN 字段 `address` 写入定制 iWAN inbound。Ricky-Hao iWAN 核心要求地址池字段为 `address_pool`。v7 采用三层防护：

1. `address` 仅作为旧配置迁移别名；
2. 稳定内部模型只输出 `address_pool`；
3. Web 提交字段不会直接复制到 sing-box JSON。

已有服务器可先运行：

```bash
python3 tools/repair_iwan_schema.py
sudo python3 tools/repair_iwan_schema.py --apply
```

脚本执行候选配置校验、备份、原子替换、重启和失败恢复。

## 2. 已实施架构

```text
Browser
  │ HTTPS / Auth / CSRF
  ▼
Caddy / Nginx
  │ 127.0.0.1:8088
  ▼
非特权 iwan-gateway Web
  │ 稳定 API 与领域模型
  │ Unix Socket / SO_PEERCRED
  ▼
root iwan-gateway-helper
  ├─ 固定服务白名单
  ├─ 固定配置路径
  ├─ sing-box check
  ├─ 文件锁与事务 ID
  ├─ 备份 / 原子替换
  ├─ 重启 / 健康检查 / 回滚
  └─ 脱敏审计日志
```

核心模块：

```text
panel/gateway/
├── application.py   非特权应用服务边界
├── runtime.py       HTTP API
├── singbox.py       稳定模型与 Ricky-Hao 渲染器
├── helper.py        root helper 与事务执行
├── client.py        Unix Socket 客户端
├── protocol.py      有界长度 JSON 协议
├── atomic.py        fsync + 原子文件替换
├── locks.py         跨进程文件锁
├── audit.py         脱敏 JSONL 审计
├── redaction.py     密码/令牌/链接脱敏
└── autosave.py      可恢复、幂等自动保存队列
```

## 3. 事务流程

所有 sing-box 保存都遵循：

```text
load current
→ migrate legacy schema
→ normalize browser payload
→ validate stable model
→ render candidate
→ acquire cross-process lock
→ sing-box check
→ timestamped backup
→ fsync + atomic replace
→ systemd restart
→ active/health check
→ commit audit event
```

任一步失败都会保留或恢复旧配置。浏览器未知字段、无效 CIDR、端口、MTU、重复节点标签、指向已删除节点的路由都会在写入前拒绝。

## 4. 权限与网络边界

- Web 进程用户：`iwan-gateway`，无 Linux capabilities；
- root helper 只接受 root 或配置的 Web UID；
- helper 在面板用户或用户组不存在时拒绝启动，而不是降级为任意本机用户可调用；
- helper 仅暴露固定操作，不接受任意命令；
- Socket 为 `0660`，运行目录允许非 root 用户进入，但不允许未授权用户打开 Socket；
- 面板默认绑定 `127.0.0.1`；
- 绑定 `0.0.0.0`/`::` 必须显式设置 `IWAN_ALLOW_PUBLIC_BIND=1`；
- 推荐使用 `deploy/caddy/Caddyfile.example`；
- Nginx 备选模板为 `deploy/nginx/iwan-gateway.conf.example`；
- 公网安全组只开放 80/443，不开放 8088。

## 5. 发布与升级

稳定发布流程：

```text
PR CI
→ 建立不可变 release/v7.1.1-source
→ 计算每个文件 SHA-256 和 Git Blob
→ 使用离线 Ed25519 私钥签署 panel-release.json
→ 安装器校验固定公钥指纹与清单签名
→ 下载固定提交文件
→ 组装和自检
→ 首次升级保存旧 systemd 服务快照
→ 创建版本目录
→ 原子切换 current 软链接
→ 启动 helper/Web
→ /healthz
→ 失败自动切回上一版本或恢复旧版服务
```

手动回滚：

```bash
bash install-panel-stable.sh --rollback
```

新架构版本位于 `/opt/iwan-gateway/releases/`，首次升级保存的旧版服务快照位于 `/opt/iwan-gateway/legacy/`。仓库只保存公钥，私钥不得进入 Git 历史。
